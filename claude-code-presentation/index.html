Checkpointing & Rewind

Your safety net ‚Äî undo anything Claude did

Every time you send a prompt, Claude Code creates a checkpoint. If something goes wrong, you can rewind.

Conversation Rewind
Go back in chat

Restore to a previous message. Useful when Claude went in the wrong direction.

Code Rewind
Revert file changes only

Undo file edits but keep the conversation. Good when the code was wrong but the discussion was useful.

Full Rewind
Revert everything

Restore both code and conversation to a previous point. Complete do-over.

üîÑ
Access via Esc Esc or /rewind

Don't be afraid to let Claude experiment. You can always rewind. Note: bash side-effects (e.g., deleting external files, API calls) cannot be undone by rewind ‚Äî it only tracks file edits and conversation.

üí°
For real version control, use Git

Rewind is great for quick undo, but git is the proper safety net for your code. Commit often, and you can always go back. Claude is excellent at managing git for you ‚Äî see the Git tab.

CLAUDE.md ‚Äî Project Memory

The CLAUDE.md file is Claude's "constitution" for your project. It's automatically read at the start of every session.

‚ö°
FIRST THING TO DO IN ANY PROJECT Run /init to create your CLAUDE.md. This is the single most impactful thing you can do for Claude Code effectiveness.

What to put in CLAUDE.md

## Overview
Automated ETL and reporting for regional sales data.

## Tech Stack
- Python 3.11+ with Polars for data processing

- EMEA region uses different tax rates

Git Concepts in 60 Seconds

Just enough to understand what Claude is doing

Repository
Your project folder

A folder tracked by git. Created with git init or cloned from a remote (like GitHub/Azure DevOps).

Commit
A save point

A snapshot of your files at a specific moment. Like "Save As" but smarter ‚Äî tracks what changed.

Branch
A parallel version

Work on something new without affecting the main version. Merge it back when done.

Push
Upload changes

Send your commits to the remote server (GitHub, Azure DevOps) so others can see them.

Pull
Download changes

Get the latest commits from the remote that others have pushed.

Diff
See what changed

Compare your current files to the last commit. Shows exactly what was added, removed, or modified.
